# 项目开发规范

## 执行规范

- **回答必须使用中文**
- **每次回答前必须先查看.windsurf/rules下的规则文档**
- **文档中的所有规则必须严格执行**
- **任务执行完成后必须将对应的 To-do 项标记为完成**

## 版本控制

- 每完成一个小节后必须进行 Git 提交（commit）
- **提交前必须确保代码编译通过**
- **提交信息应清晰描述完成的内容**
- **提交信息强制使用中文**
- **提交确认规则**：
  - **无 To-dos 任务列表**：当只是执行小修改时（没有创建 To-dos 任务列表），Git 提交前必须先询问用户确认是否提交
  - **有 To-dos 任务列表**：当有 To-dos 任务列表进行多项任务执行时，完成任务后直接 Git 提交，无需用户确认
- 中文提交方法：
  1. 使用 `write` 工具创建 `commit-msg.txt` 文件（确保 UTF-8 编码）
  2. 执行 `git commit -F commit-msg.txt` 
  3. 清理临时文件 `Remove-Item commit-msg.txt`
  4. **注意**：不要使用 PowerShell 的 `Out-File` 或字符串变量创建文件，会导致编码问题

## 编译验证

- **确认无编译错误后方可提交代码**
- **编译失败时需修复所有错误后再提交**

## 开发规范

- 严格遵守 vue3 开发规范
- 代码风格、命名规则、架构设计均需符合规范要求
- 尽量多拆分组件，提高代码复用性和可维护性
- 组件设计要遵循单一职责原则，避免过于复杂的组件
- 尽量多使用公共组件，减少重复代码
- 保持组件的轻量级和可复用性
- 每个页面代码尽量简洁，避免过度复杂，最好不要超过500行
- 代码逻辑要清晰，注释要充分
- 代码要易于阅读和维护

## 代码简化原则

1. 保留功能：只改变实现方式，不改变行为
2. 增强清晰度：减少不必要的复杂度和嵌套
3. 消除冗余：删除重复代码和无用注释
4. 保持平衡：避免过度简化导致可读性下降
5. 聚焦范围：优化最近修改的代码

## 任务跟踪

- 使用 To-do 工具跟踪复杂任务的完成情况
- 每完成一个 To-do 项，必须立即标记为完成
- 确保任务完成状态与实际进度同步


**注意**：所有步骤必须按序执行，确保每一步都通过验证后再进行下一步。
